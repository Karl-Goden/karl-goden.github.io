Section 0: Flow

torch.compile receive a plain function or torch.nn.Module or torch.fx.GraphModule (which is also a torch.nn.Module).
It calls dynamo which use python frame evaluation to trace the plain function or torch.nn.Module's forward function's
control flow and data flow. torch.export use example inputs to generate a control flow flattened graph ExportedProgram
which is used by other ai compiler like tensorrt, tvm, etc. For its own builtin ai compiler inductor, dynamo will generate
fx data flow graph and use inductor to accelerated hardware code. torch operation is represented by aten. If inductor does
not support certain aten operation, it will split the data flow graph and use torch's eager mode to execute it by aten library.
TorchScript is in jit directory which use aten library to execute operation. torch.jit.script(model).save can save the model
in torchscript format file. torch.save can save the model in torch's pickle format file.

Weired Name

aten: a tensor library.
c10: core c++ library: data type, device abstraction, math library (softfloat, complex).
fx: functional transformation for data flow graph.
distribted & distribution: distributed training and probability distribution.

Section 1: Tensor

_tensor.py: python tensor definition.

Section 2: ATen

_ops.py: python binding of aten operation.
ATen/native/native_functions.yaml declare all aten operation signatures and dispatch to different backend implementation based
on which torchgen can generate its declaration and definition code.

Section 3: Dynamo

Dynamo is a python level JIT compiler. It use python frame evaluation to trace the control flow and data flow of the
forward function of a torch.nn.Module or a plain function. It can generate fx data flow graph or export program graph.

Section 4: Inductor

Inductor is a backend compiler for pytorch. It has several main components: graph capture, graph optimization, kernel
scheduling, code generation, and runtime execution.

Section 5: Fx

Fx is a functional transformation for pytorch programs. It has three main components: tracer, graph, and interpreter.
The tracer records operations performed on tensors and builds a graph representation of the computation.
The graph is a data structure that represents the computation as a series of nodes and edges.
The interpreter executes the graph by traversing its nodes and performing the corresponding operations.

Section 6: Func

Func is a functional programming library for pytorch. It provides a set of higher-order functions that can be used to
manipulate tensors and build complex computations in a functional style.
